Bug Fixes - Sprint 4
====================

Bug 1: Duplicate meal assignment on the same day (meal-planner.js)
- Issue: Users could assign the same recipe to multiple meal types on the same day.
- Fix: Added a duplicate check before saving. If the recipe is already assigned to another meal slot on that day, an error message is shown.

Bug 2: NaN displayed for recipe cost (recipes.js)
- Issue: When a recipe had no cost value (null/undefined), "$NaN" was displayed on the recipe card.
- Fix: Added a null check so missing cost defaults to "$0.00".

Bug 3: Unsanitized search query in recipe filter (recipes.js)
- Issue: Special characters (%, _, \) in the search input were passed directly into the ILIKE filter, causing broken or unintended query results.
- Fix: Escaped special LIKE characters before building the Supabase query.

Bug 4: NaN sent to database for empty recipe time (recipes.js)
- Issue: Leaving the time field empty when creating/editing a recipe sent NaN to the database (parseInt of empty string).
- Fix: Added a check to send null instead of NaN when the time field is empty.

Bug 5: Password validation mismatch on register page (register.html)
- Issue: The UI label said "8+ chars" but the code only enforced a minimum of 6 characters.
- Fix: Updated the validation to require at least 8 characters, matching the displayed requirement.
